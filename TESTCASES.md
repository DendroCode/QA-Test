# Тест-кейсы для микросервиса

---

## test_1. Создание объявления  

### Шаги:
1. Отправить POST-запрос на https://qa-internship.avito.com/api/1/item с тестовыми данными.
```
Для примера можно отправить такой запрос:

{
  "sellerID": 592731,
  "name": "Test",
  "price": 100,
  "statistics": {
    "likes": 1,
    "viewCount": 2,
    "contacts": 3
  }
}
```
2. Проверить статус-код ответа.
3. Проверить наличие ID в ответе от сервера. 

### Ожидаемый результат:
1. Статус-код 200.
2. В ответе от сервера получен ID объявления.

## test_2. Получить объявление по идентификатору

### Шаги:
1. Отправить GET-запрос на https://qa-internship.avito.com/api/1/id, где id — идентификатор из ранее созданного объявления. 
```
Пример: 165b8587-aae7-406f-b053-77a6689a9ba4
```
2. Проверить статус-код ответа. 
3. Проверить структуру полученного ответа. 

### Ожидаемый результат:
1. Статус-код 200. 
2. В ответе от сервера есть поля: createdAt, id, sellerId, name, price, statistics (с вложенными contacts, likes, viewCount).
3. Ответ от сервера с полем id соответствует тому, который указан в GET-запросе. 

## test_3. Получить все объявления по идентификатору продавца

### Шаги:
1. Отправить GET-запрос на https://qa-internship.avito.com/api/1/sellerId/item, где sellerId — идентификатор продавца.
```
Пример: 592731
```
2. Проверить статус-код ответа.
3. Проверить полученные от сервера объявления. 

### Ожидаемый результат: 
1. Статус-код 200.
2. В ответе от сервера есть поля: createdAt, id, sellerId, name, price, statistics (с вложенными contacts, likes, viewCount).
3. Ответ может содержать несколько объявлений перечисленных через запятую. Допустимы неуникальные значения полей. 
4. В каждом объявлении значение sellerId соответствует sellerID, указанному в GET-запросе.

## test_4. Получить статистику по id предмета. 

### Шаги:
1. Отправить GET-запрос на https://qa-internship.avito.com/api/1/statistic/id, где id — идентификатор из объявления.
```
Пример: 165b8587-aae7-406f-b053-77a6689a9ba4
```
2. Проверить статус-код. 
3. Проверить, что ответ содержит статистику по объявлению.

### Ожидаемый результат: 
1. Статус-код 200.
2. Ответ от сервера содержит 3 поля: contacts, likes и viewCount с присвоенными значениями. 
3. Все значения у полей указаны в виде целых чисел.
