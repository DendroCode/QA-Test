# Баг-репорты

## ID_001

### Приоритет: 
High

### Заголовок: 
Отрицательные значения в карточке товара.

### Описание: 
Сервер успешно создаёт объявления с отрицательной ценой и/или статистикой. 

### Шаги воспроизведения: 
1. Отправить POST-запрос на https://qa-internship.avito.com/api/1/item с отрицательными параметрами. Пример:
```
{
  "sellerId: 592731,
  "name": "buggy",
  "price": -50,
  "statistics": {
    "likes": 1,
    "viewCount": -2,
    "contacts": -9
  }
}
```
2. Записать id объявления. Проверить создание по GET-запросу на https://qa-internship.avito.com/api/1/item/408228de-8d04-417f-9d87-36f93529e431

### Ожидаемый результат:
На этапе POST-запроса появится ошибка (400 Bad Request), запрещающая создание объявлений с отрицательными значениями. 

### Фактический результат:  
Объявление успешно создаётся с кодом-статусом 200. Полученный GET-запрос выглядит так:
```
[
    {
        "createdAt": "2025-12-29 19:27:36.950294 +0300 +0300",
        "id": "408228de-8d04-417f-9d87-36f93529e431",
        "name": "buggy",
        "price": -50,
        "sellerId": 592731,
        "statistics": {
            "contacts": -9,
            "likes": 1,
            "viewCount": -2
        }
    }
]
```

## ID_002

### Приоритет: 
Medium

### Заголовок: 
Пустое имя в объявлении 

### Описание: 
Сервер создаёт объявления с именем, состоящим только из пробелов.


### Шаги воспроизведения: 
1. Отправить POST-запрос на https://qa-internship.avito.com/api/1/item с пробелами в имени. Пример: 
```
{
  "sellerId": 592731,
  "name": " ",
  "price": 1,
  "statistics": {
    "likes": 4,
    "viewCount": 5,
    "contacts": 67
  }
}
```
2. Записать id объявления. Проверить создание по GET-запросу на https://qa-internship.avito.com/api/1/item/86b38e1a-a98f-4e6f-9c68-80925efb5775

### Ожидаемый результат:
Сервер вернёт ошибку (400 Bad Request) и попросит дать корректное имя. 

### Фактический результат:  
Сервер принимает запрос и создаёт объявление без имени.

## ID_003

### Приоритет: 
High 

### Заголовок: 
Ошибка при создании объявления с нулевой статистикой. 

### Описание: 
Сервер не позволяет создать объявление со статистикой 0 у одного из параметров (лайки, просмотры или контакты).

### Шаги воспроизведения: 
1. Отправить POST-запрос на https://qa-internship.avito.com/api/1/item с нулевой статистикой. Пример:
```
{
  "sellerId": 592731,
  "name": " ",
  "price": 1,
  "statistics": {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
}
```

### Ожидаемый результат:
Сервер примет нулевую статистику, так как новый товар изначально имеет нулевые показатели.

### Фактический результат:  
Сервер выдаёт код-статус 400 с сообщением «поле likes обязательно».
> Если в likes будет ненулевое значение, сервер так же вернёт код-статус 400, но с сообщением «поле viewCount обязательно». Такая же логика распространяется на поле contacts.
